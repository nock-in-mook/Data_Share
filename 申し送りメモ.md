# フォルダ大移動作戦 - 申し送りメモ

## 実施日: 2026-03-07

## 概要
Dropbox直下のフォルダ名を、クロスプラットフォーム互換性のためにリネームした。
macOSでドット始まりが隠しフォルダになる問題、日本語・特殊記号がトラブルを起こす問題を解決するため。

## 変更内容

### フォルダリネーム
| 旧名 | 新名 |
|---|---|
| `.★自作アプリ2026-★` | `_Apps2026` |
| `.claude-sync` | `_claude-sync` |

### シンボリックリンク（`C:\Users\msp\.claude\` 配下）
全て junction で `_claude-sync` 内を参照するように再作成済み：
- `CLAUDE.md` → `_claude-sync/CLAUDE.md`
- `settings.json` → `_claude-sync/settings.json`
- `commands/` → `_claude-sync/commands/`
- `memory/` → `_claude-sync/memory/`
- `projects/` → `_claude-sync/projects/`

### コード修正（旧パス → 新パスへの書き換え）
1. **`_claude-sync/setup.bat`** — `.claude-sync` → `_claude-sync` に全置換
2. **`_Apps2026/Data_Share/client/data_share_client.py`** (39行目) — `APPS_ROOT` パスを `_Apps2026` に変更済み
3. **`_Apps2026/Data_Share/.claude/settings.local.json`** — 旧パス参照を `_Apps2026` に修正
4. **`_Apps2026/透明キーボード/transparent_keyboard.py`** (227行目) — `apps_dir` を `_Apps2026` に修正
5. **`_Apps2026/everyWEAR/.claude/settings.local.json`** — 旧パス参照を `_Apps2026` に修正（Dropbox同期で自動適用）

### ショートカット再作成
- **即シェア君**: スタートメニュー + スタートアップ（`setup_shortcuts.ps1` で再作成）
- **透明キーボード**: スタートメニュー + デスクトップ（PowerShellで再作成）

### 即シェア君リビルド
- 旧exeが `D:/Dropbox/.★自作アプリ2026-★/images/` と `text/` を自動生成していた（39-41行目のハードコードパス）
- ソース修正後に PyInstaller でリビルド実施
- `client/dist/RapidShare.exe` と `インストール用EXEファイル/RapidShare.exe` の両方を更新済み

## 旧フォルダについて
- `D:\Dropbox\.★自作アプリ2026-★` はダミー（このセッションのCWDのために存在）
- セッション終了後に手動で削除する予定
- 中身は空（Data_Share/ だけ残っているがCWD参照のため）

## 他のPCでの対応
1. `D:\Dropbox\_claude-sync\setup.bat` をダブルクリック（シンボリックリンク再作成）
2. 即シェア君・透明キーボードのショートカットが壊れていたら再インストール
   - 即シェア君: `_Apps2026/Data_Share/インストール用EXEファイル/install.bat` を実行
   - 透明キーボード: `_Apps2026/透明キーボード/install_keyboard.bat` を実行

## 修正漏れチェック済み
- `_Apps2026` 配下の全 `.py`, `.bat`, `.sh`, `.json`, `.ps1`, `.ts` を grep で確認
- SESSION_LOG/HISTORY（過去ログ）以外に旧パス参照は残っていない
- `KANJI_HANTEI/benchmark_results/` のJSON（過去データ）に旧パスあるが、書き換え不要

### transcript_export.py の旧パス対応
- `_claude-sync/transcript_export.py` がセッションログ出力時に jsonl 内の `cwd`（旧パス）をそのまま使って旧フォルダにファイルを生成していた
- `normalize_cwd()` 関数を追加し、旧パス→新パスの自動変換ロジックを実装
- `determine_output_path()` と `archive_session()` の両方に適用済み
- これにより、旧パスで記録されたセッションのログ出力でも旧フォルダが復活しなくなった

## 注意事項
- 旧セッション（旧パスで作成されたもの）は開けない（パスが変わったため）
- SESSION_HISTORY.md に過去ログは蓄積されているので内容は失われていない
